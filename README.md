# 游戏制作人模拟器 (Game Dev Simulator)

一款融合 **卡牌肉鸽** 与 **模拟经营** 的 Web 游戏。你将扮演一个独立游戏工作室的负责人，管理团队、分配工作、积累卡牌，并在每周/月末的 Boss 战中用构筑的牌组击败强敌。

## 快速开始

```bash
# 安装依赖
npm install

# 开发模式运行
npm run dev

# 构建生产版本
npm run build
```

构建产物在 `dist/` 目录，双击 `启动游戏.bat` 即可在浏览器中运行（无需开发环境）。

## 技术栈

| 技术                | 用途                   |
| ------------------- | ---------------------- |
| TypeScript (strict) | 类型安全的开发语言     |
| Vite 8              | 构建与开发服务器       |
| Phaser 3.90         | 战斗场景的 Canvas 渲染 |
| Immer               | 不可变状态管理         |
| Web Audio API       | 程序化音效生成         |

采用 **DOM + Canvas 混合架构**：经营界面使用 DOM 渲染，战斗场景使用 Phaser Canvas 渲染。

## 核心玩法

```
标题画面 → 难度选择 → 初始员工选择 → 项目开始
  │
  ├─ 日常经营（每月30天，每天3个时间段）
  │   选择活动 → 推进进度 → 获得卡牌 → 消耗体力
  │
  ├─ 每周战斗（第7/14/21天）
  │   用积累的卡牌对战普通敌人 → 胜利获得装备
  │
  ├─ 月末Boss战（第30天）
  │   锁定牌组 → 挑战Boss → 胜利获得Buff奖励
  │
  ├─ 发布游戏（三维进度均达100%时可选）
  │   标记发布 → 最终Boss（难度x2）→ 项目结算
  │
  └─ 新项目循环
      商店购物 → 招募员工 → 开始下一个项目
      （资金、员工、装备、道具跨项目保留）
```

## 系统详解

### 1. 日常经营系统

每天分为 **早/下/晚** 三个时间段，每个时间段可选择一项活动：

| 活动     | 体力 | 效果                    | 产出卡牌     |
| -------- | ---- | ----------------------- | ------------ |
| 编程开发 | 12   | 编程进度+4              | 攻击牌       |
| 美术制作 | 12   | 美术进度+4              | 技能牌       |
| 开会讨论 | 6    | 策划进度+3，士气+2      | 平衡型卡牌   |
| 头脑风暴 | 14   | 创意+15，策划+2         | 高稀有度卡牌 |
| 团建活动 | 18   | 士气+12，忠诚+5         | -            |
| 加班赶工 | 22   | 编程+7，美术+5，士气-10 | 含负面状态牌 |
| 休息调整 | 0    | 体力+20，移除负面牌     | -            |

活动效果受员工职位影响，程序员做编程消耗更低，美术做美术更高效。

### 2. 战斗系统

类似《杀戮尖塔》的回合制卡牌战斗：

- **每回合**：抽5张牌，3点能量，打出卡牌造成效果
- **卡牌类型**：攻击牌（伤害）、技能牌（护甲/抽牌）、能力牌（持续被动）、状态牌（负面效果）
- **加成计算**：力量Buff增加攻击伤害，敏捷Buff增加技能护甲，灵感Buff使下张攻击伤害翻倍
- **减益效果**：虚弱(-25%伤害)、易伤(+50%受伤)

**敌人AI** 按预设的行为序列循环执行攻击、防御、增益、削弱等动作，Boss 每回合额外获得+1力量。

**难度缩放**：敌人强度随月份和周数递增，最终Boss在普通Boss基础上再乘以2倍。

### 3. 卡牌系统

共 **34张** 卡牌，分4类：

| 类型   | 数量 | 代表卡牌                                                                |
| ------ | ---- | ----------------------------------------------------------------------- |
| 攻击牌 | 10   | 代码冲刺(6伤害)、Debug(8伤害)、重构代码(10伤害+抽1)                     |
| 技能牌 | 10   | 代码审查(7护甲)、视觉打磨(4护甲+抽1)、头脑风暴(抽3)                     |
| 能力牌 | 5    | 每日站会(每回合抽1)、自动化流水线(每回合+3护甲)、敏捷开发(每回合+1能量) |
| 状态牌 | 9    | 技术债(不可打出)、疲劳(每回合-2HP)                                      |

卡牌通过日常活动获得，稀有度分 COMMON / UNCOMMON / RARE / LEGENDARY。

### 4. 员工系统

**18名** 可招募员工，分程序、美术、策划、QA等职位：

- **属性**：编程/美术/策划/创意能力值、体力、士气、忠诚度
- **星级**：2-5星，决定基础能力
- **特质**：可见特质和隐藏特质（满足条件后揭示）
- **贡献卡牌**：每位员工关联特定卡牌，招募后加入牌库

初始选择3-5名员工组队，后续项目间可在招募阶段扩充团队。

### 5. 装备系统

**10件** 装备，通过Boss战胜利后三选一获得，永久生效：

| 类别 | 装备                           | 效果                              |
| ---- | ------------------------------ | --------------------------------- |
| 防御 | 铁甲 / 钢板甲 / 钻石盾         | +5/+10/+15 战斗初始护甲           |
| 攻击 | 锋利小刀 / 勇者之刃 / 传说之剑 | +1/+2/+4 攻击伤害                 |
| 恢复 | 草药包 / 治愈水晶              | 每回合+2/+4 HP                    |
| 能量 | 功能饮料 / 咖啡机              | 最大能量+1，咖啡机额外每回合抽1张 |

装备可在商店用资金购买，也可通过战斗奖励获得。

### 6. Buff系统

**34个** Buff效果，分三类：

- **增益 (POSITIVE)**：力量、敏捷、每日站会、流水线自动化等
- **减益 (NEGATIVE)**：虚弱、易伤、疲劳
- **隐蔽 (HIDDEN)**：满足特定条件自动激活（如"草台班子"：全员应届生时触发，敌人强度+15%）

Buff来源包括：Boss战奖励、事件选择、卡牌效果、装备被动。

### 7. 消耗品系统

**8种** 道具，在商店购买，日常阶段使用：

| 道具                | 价格       | 效果               |
| ------------------- | ---------- | ------------------ |
| 急救包 / 高级急救包 | 500 / 1200 | 项目HP +15 / +30   |
| 能量棒 / 精品咖啡   | 400 / 1000 | 全员体力 +25 / +50 |
| 防护卷轴            | 800        | 下场战斗 +20 护甲  |
| 激励演讲稿          | 600        | 全员士气 +15       |
| 调试工具箱          | 1500       | 移除1个负面效果    |
| 灵感之球            | 2000       | 创意+20，品质+5    |

### 8. 事件系统

**5个** 预设剧情事件，按时间线或概率触发：

| 事件          | 触发条件         | 内容                             |
| ------------- | ---------------- | -------------------------------- |
| 工作室第一天  | 第1月第1天       | 选择：鼓舞士气 或 精打细算       |
| 第一个严重Bug | 第1月第10天(80%) | 选择：加班修复 或 延后处理       |
| 版号寒冬      | 第2月第1天       | 选择：士气-5 / 转向出海 / 缩编   |
| 电竞世界赛    | 第2月(60%)       | 选择：聚众看比赛 或 禁止         |
| 投资人来访    | 第3月(70%)       | 选择：大融资 / 小融资 / 保持独立 |

每个选项带来不同的资金、士气、Buff变化。

### 9. 音效系统

使用 **Web Audio API** 程序化生成所有音效，无需额外音频文件：

- 点击反馈、攻击音效、受击音效
- 治疗、Buff、格挡音效
- 出牌/抽牌音效
- 回合开始提示音
- 胜利/失败琶音序列
- BGM 背景音乐（mp3文件循环播放）

页面不可见时自动暂停，可见时恢复。

### 10. 存档系统

- 基于 **localStorage** 的自动存档
- 支持存档版本迁移（新增字段自动补全默认值）
- 确定性随机数种子，存档可精确复现

## 架构设计

```
┌──────────────────────────────────────────┐
│              应用入口 (main.ts)           │
├──────────────────────────────────────────┤
│  内核层: GameKernel + GameState + EventBus│
│  ─ Immer 不可变状态，EventBus 事件解耦   │
├──────────────────────────────────────────┤
│  数据层: JSON 数据文件 + 类型化模型       │
│  ─ 全部游戏内容由 JSON 驱动              │
├──────────────────────────────────────────┤
│  逻辑层: DailySystem / CombatManager /   │
│         EnemyScaler / EventSystem        │
├──────────────────────────────────────────┤
│  渲染层: DOM 界面 (UIManager + 12 Screen)│
│         Canvas 战斗 (Phaser CombatScene) │
├──────────────────────────────────────────┤
│  持久层: localStorage 存档 + 版本迁移     │
└──────────────────────────────────────────┘
```

核心设计原则：

- **数据驱动**：所有游戏内容通过 JSON 配置，无需改代码即可调整平衡性
- **事件解耦**：系统间通过 EventBus 通信，40+ 事件类型覆盖全流程
- **不可变状态**：使用 Immer 的 `produce()` 保证状态一致性
- **确定性随机**：种子驱动的 RNG，存档可精确复现
